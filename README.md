# 📊 Sales Database Case Study

This project is a **real-world SQL case study** built using MySQL (MariaDB via XAMPP).

It focuses on designing a relational database and solving business-driven queries using SQL.

---

## 🧠 Objective

* Understand real-world database structure
* Perform data analysis using SQL
* Solve business problems using queries
* Practice joins, aggregations, and filtering

---

## 🗂 Database Structure

The database consists of the following tables:

* **products** → product details
* **customers** → customer information
* **employees** → employee data
* **sales_order** → order transactions

---

## ⚙️ Concepts Used

* INNER JOIN, LEFT JOIN
* GROUP BY & HAVING
* Aggregate functions (SUM, AVG, COUNT)
* Filtering using WHERE
* Subqueries
* CASE-based problem solving

---

## 📌 Key Business Problems Solved

* Total products sold
* Orders by delivery status
* Customer-wise purchase analysis
* Daily revenue calculation
* Employee performance tracking
* High-value order identification
* Customers with no purchases
* Orders above specific thresholds

---

## 🛠 Tools Used

* MySQL
* MariaDB (XAMPP)
